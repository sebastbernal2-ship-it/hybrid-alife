# Visual Gallery

A guide to every plot and visual artifact this project can produce, what each
one means, and the exact command that generates it. Plots are written by
`scripts/generate_report.py` and the helpers in `src/hybrid_alife/viz/plots.py`.

Quick start (one command produces every plot in this gallery):

```bash
python scripts/run_sim.py --config configs/smoke200.yaml
python scripts/generate_report.py outputs/runs/smoke200
```

After that, the four core plots live in `outputs/runs/smoke200/plots/`:

| Plot | File | Source artifact |
| --- | --- | --- |
| World fields | `plots/world_fields.png` | `checkpoint_final.pkl` |
| Agent positions | `plots/agents.png` | `checkpoint_final.pkl` |
| Metrics over time | `plots/metrics.png` | `metrics.jsonl` |
| MAP-Elites archive | `plots/map_elites.png` | `map_elites.npz` |

If a source artifact is missing the report falls back gracefully and emits
`_<name>: missing source <path>_` in `report.md` — you can still produce the
rest.

---

## 1. World fields — `plots/world_fields.png`

**What it is.** A grid of 2D heatmaps over the shared world, one panel per
proxy-physics channel: terrain bias / ridge / basin / channel-mask, resources,
hazards, flow (vector magnitude), shear, shear-gradient, inertial-lift proxy,
margination/enrichment, curvature, concentration, and metabolites.

**What it means.** This is the sensory ecology agents actually inhabit. Use it
to confirm the world is non-trivial: structured curvature/shear creates
microfluidic-inspired pressure for evolution; a flat field would mean the
"sixth-sense" channels carry no information (a control you can run via
`configs/ablation_uniform_field.yaml`).

**Generated by.** `hybrid_alife.viz.plots.plot_world_fields`, called from
`scripts/generate_report.py`.

**Reproduce.**

```bash
python scripts/run_sim.py --config configs/smoke200.yaml
python scripts/generate_report.py outputs/runs/smoke200
# → outputs/runs/smoke200/plots/world_fields.png
```

---

## 2. Agent positions — `plots/agents.png`

**What it is.** A scatter of every alive embodied-branch agent at the final
checkpoint, overlaid on a backdrop of one world field (typically channel mask
or enrichment).

**What it means.** Quick visual sanity check on emergent spatial behavior:
clustering near curvature ridges, channels, or enrichment hotspots is the
microfluidic-separation analog made visible. A fully uniform scatter under a
structured world is suspicious and worth investigating.

**Generated by.** `hybrid_alife.viz.plots.plot_agent_positions`.

**Reproduce.** Same command as §1; produced from `checkpoint_final.pkl`.

---

## 3. Metrics over time — `plots/metrics.png`

**What it is.** Line plots over generations / steps for the core metrics
written into `metrics.jsonl`: survival fraction (per branch), action entropy,
communication usage / message energy, coordinated-behavior index, mean Avida
merit, enrichment separation index, and lineage depth.

**What it means.** Your primary dashboard. Expected qualitative signatures on
the 200-generation smoke (see README *Current results*): embodied alive
fraction ≈ 1.0 (reseeding healthy), action entropy ≈ 1.6 / 2.3 (diverse
behavior), CBI ≈ 0.7 (shared action distributions), Avida mean merit climbing
from ~1.0 to ~7.5 (replicators accumulating useful work). Flat or collapsing
curves are the first thing to look at when a run looks wrong.

**Generated by.** `hybrid_alife.viz.plots.plot_metrics`.

**Reproduce.** Same command as §1; produced from `metrics.jsonl`.

---

## 4. MAP-Elites archive — `plots/map_elites.png`

**What it is.** A 2D heatmap of the MAP-Elites behavior archive: each cell is
a behavior-descriptor bin, color encodes the elite fitness in that cell.
Empty cells are uncovered behaviors.

**What it means.** A direct view of behavioral diversity. *Coverage* (fraction
of non-empty cells) is the quality-diversity headline number — the smoke run
sits at ≈ 0.11, sparse but non-zero. A run that fills more of the grid has
discovered a wider range of behaviors; a run that collapses to a single bright
cell has converged.

**Generated by.** `hybrid_alife.viz.plots.plot_map_elites`. Requires
`map_elites.npz` (only produced by configs that enable the archive, e.g.
`smoke200.yaml`).

**Reproduce.** Same command as §1; produced from `map_elites.npz`.

## 5. Ablation matrix summary (markdown, not a plot)

**What it is.** A side-by-side comparison of headline metrics across configs,
with median / IQR / Cliff's δ vs baseline. Written as markdown, not as a PNG.

**What it means.** The cleanest way to claim that a structural component (the
six "sixth-sense" channels, communication, world drift) is *doing measurable
work*. If `ablation_no_comms` or `ablation_uniform_field` matches baseline on
every headline metric, that component is decorative.

**Reproduce.**

```bash
python scripts/run_ablation_matrix.py \
    --configs configs/smoke200.yaml configs/ablation_no_comms.yaml \
              configs/ablation_static_world.yaml configs/ablation_uniform_field.yaml \
    --seeds 3 --include-shadow \
    --out-dir outputs/ablation_matrix
# → outputs/ablation_matrix/summary.md, plus per-config report.md
```

## 6. Per-run auto-generated report — `report.md`

**What it is.** The markdown wrapper that embeds the four plots above plus
headline-metrics tables, optional baseline deltas, and a plots section that
degrades gracefully when artifacts are missing.

**Useful flags.**

- `--no-plots` — skip plot generation entirely (CI-friendly, no matplotlib).
- `--headline` — emit only the headline-metrics table (for sweep aggregation).
- `--baseline <run_dir>` — add per-metric Δ vs another run.
- `--out <filename>` — choose a non-default output filename.

**Reproduce.**

```bash
python scripts/generate_report.py outputs/runs/smoke200
python scripts/generate_report.py outputs/runs/smoke200 --no-plots --headline
python scripts/generate_report.py outputs/runs/smoke200 \
    --baseline outputs/runs/ablation_no_comms
```

```bash
make gallery-clean  # remove outputs/runs/smoke200/plots/
```

## Troubleshooting

- **`matplotlib` import errors** — install the dev extras: `pip install -e ".[dev]"`,
  or use `--no-plots` to produce a report without figures.
- **`plots/<name>.png` is missing and the report says "missing source"** — the
  upstream artifact for that plot wasn't produced. Check that your config
  enables MAP-Elites for `map_elites.png`, and that the run completed long
  enough to write `checkpoint_final.pkl`.
- **Plots render but look flat** — confirm you aren't on
  `configs/ablation_uniform_field.yaml` (intentionally flat) or
  `ablation_blind.yaml` (intentionally degenerate).
- **CI fast path** — `python scripts/generate_report.py <run> --no-plots`
  produces the markdown report without importing matplotlib.
